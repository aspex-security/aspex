# Code of Conduct

## Our commitment

Aspex is a security-focused open-source project. We are committed to maintaining a space where contributors can focus on the work: finding and documenting real security risks in the MCP ecosystem, building detection rules, and improving the tools.

We expect everyone participating in this project (issues, pull requests, discussions, community spaces) to be direct, respectful, and focused on the work.

---

## Expected behavior

- Be direct and specific. Say what you mean. Vague or passive-aggressive feedback is harder to act on than clear, specific feedback.
- Critique code and ideas, not people. "This detection logic will produce false positives because..." is constructive. Personal attacks are not.
- Assume good intent. Most contributors are here because they care about MCP security. Start from that assumption.
- Be patient with new contributors. Not everyone has a security background or knows Go. Good explanations compound over time.
- Respect the project scope. This is a security tool with a defined OSS/SaaS line. Contributions that blur that line will be declined, but that is a scope decision, not a personal one.

---

## Unacceptable behavior

- Personal attacks, insults, or harassment in any form.
- Discriminatory language or behavior based on identity.
- Publishing others' private information without consent.
- Deliberate intimidation or sustained disruption of discussion.
- Using this project's issues, PRs, or discussions to promote unrelated products or services.

---

## Scope

This code of conduct applies in all project spaces: GitHub issues, pull requests, discussions, and any official community channels associated with this project. It also applies when someone is representing the project in public spaces.

---

## Enforcement

Violations may be reported to [steven.d@onyx.security](mailto:steven.d@onyx.security). All reports will be reviewed and investigated. The project maintainers will respond in the way they judge appropriate given the circumstances, up to and including permanent removal from the project.

Maintainers who do not follow or enforce this code of conduct may face temporary or permanent consequences as determined by other project leads.
